# LoveTI 项目架构设计

## 系统工程视角总结

### 模块拆解
1. **题目引擎** - 题库管理、分发、维度映射
2. **计算核心** - ABCD频次统计、全排列生成、序列判定
3. **结果生成器** - 人格档案库、文案渲染、标签提取
4. **可视化层** - 雷达图、序列条、动画控制
5. **前端展示** - 题目录入、进度指示、结果页

### 模块交互关系
```
用户交互 → 题目引擎 → 计算核心 → 结果生成器 → 可视化层 → 前端展示
数据逆向回流：人格档案JSON → 图表数据 → 页面渲染
```

### 实现方式
- **前端**: React 18 + TypeScript + Tailwind CSS + Framer Motion + Recharts
- **状态管理**: Zustand
- **构建**: Vite
- **数据**: 静态JSON文件（题目+人格档案）

### 技术依赖
| 类别 | 技术 |
|------|------|
| UI框架 | React 18+ |
| 类型安全 | TypeScript 5.x |
| 样式 | Tailwind CSS 3.x |
| 动画 | Framer Motion 11.x |
| 图表 | Recharts 2.x |
| 状态 | Zustand 4.x |
| 构建 | Vite 5.x |

### 瓶颈与难点
1. **序列冲突**: ABCD频次相等时的唯一序列判定
2. **末位区分**: CBAD vs CBDA等相近序列的精确区分
3. **中性化题目**: 避免所有性别刻板印象词汇
4. **人格唯一性**: 24种人格必须有明显可感知差异
5. **心理测量学**: 信度、效度、社会期望偏差控制

### 整体串联逻辑
```
[START] → [LOADING] → [ANSWERING] → [CALCULATING] → [RESULT]
                              ↓            ↓
                         [PROGRESS]    [ANIMATING]
```

### 核心算法
- 全序列判定: 4! = 24种唯一排列，基于频次降序+字典序解决冲突
- 末位区分: 以第四位维度作为"浪漫盲区"的核心区分点
